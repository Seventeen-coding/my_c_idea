## 单例模式 Singleton Pattern

> 针对生成对象提供设计思想，提供唯一一个对象
>

### 解决问题

多处地方同时申请对象时，只返回唯一一个对象，这样好处是省空间，加速调用速度。

例如：

工具类（log），其目的是方便，快速达到目的， 假如多处地方调用，经常申请空间和生成对象花很多时间，从而会减慢速度，单个的话只会申请一次空间，每次调用立马返回对象。
